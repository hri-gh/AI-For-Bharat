## Key Design Decisions

### 1. Hybrid AI Approach
**Why combine OCR + CLIP?**
- OCR is precise for text-heavy screenshots (code, errors, chats)
- CLIP understands visual context (memes, designs, diagrams)
- Combining both gives best accuracy across all screenshot types

### 2. Lazy Loading
**Why not load models at startup?**
- Saves 5-10 seconds if no images found
- Reduces memory usage
- Better user experience (faster feedback)

### 3. Configurable Categories
**Why JSON config instead of hardcoded?**
- Users have different needs (gaming, work, school)
- Easy to customize without code changes
- Shareable configurations

### 4. Watch Mode
**Why continuous monitoring?**
- Automates daily workflow
- Organizes screenshots as they're taken
- No manual intervention needed

### 5. Dual Deployment
**Why support both local and Colab?**
- Colab: Easy testing, free GPU, no setup
- Local: Daily automation, privacy, offline use
- Covers all user scenarios
